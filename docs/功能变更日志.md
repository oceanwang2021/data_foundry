# 功能变更日志

## 2026-05-11 采集参数表 SQL 导入

### 变更内容

- 在“数据范围 / 采集参数表”中新增 `SQL 导入`方式。
- 参数导入现在分为两种互斥模式：
  - `手动/文件`：手动新增、编辑、粘贴 Excel 表格、CSV/XLSX 文件导入。
  - `SQL 导入`：输入 SELECT SQL 查询目标表数据，并将查询结果填充为采集参数行。
- SQL 导入成功后会覆盖当前参数行，并清除文件导入草稿，避免多种来源混用。
- 后端仅允许执行单条 `SELECT` 查询，并限制最多返回 1000 行。
- SQL 返回字段会校验参数字段名，必须通过 `as` 别名与采集参数字段保持一致。

### 使用方式

1. 进入需求定义页面的“数据范围”区域。
2. 在“采集参数表”中切换到 `SQL 导入`。
3. 输入查询 SQL，字段别名需要和参数字段一致。
4. 点击 `执行导入`。
5. 导入成功后，下方参数表会显示 SQL 查询结果生成的参数行。

示例：

```sql
select distinct
  COMNAME as COMNAME,
  COMCODE as COMCODE
from IR_ADAS_COMPUTE_CONFIG
```

说明：

- 如果参数字段是 `COMNAME`、`COMCODE`，SQL 返回列也必须叫 `COMNAME`、`COMCODE`。
- 如果缺少字段，页面会提示缺少哪些参数列。
- 当前 SQL 导入只作为参数表填充方式，不额外保存 SQL 来源配置。
