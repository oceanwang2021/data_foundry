# 功能变更日志

## 2026-05-11 采集参数表 SQL 导入

### 变更内容

- 在“数据范围 / 采集参数表”中新增 `SQL 导入`方式。
- 参数来源现在可选择两种方式：
  - `手动/文件`：手动新增、编辑、粘贴 Excel 表格、CSV/XLSX 文件导入，保存为固定参数行。
  - `SQL 导入`：保存 SELECT SQL 作为动态参数来源，预览和任务生成时再查询最新参数行。
- 从 `手动/文件` 切换到 `SQL 导入` 时，会清除文件导入草稿，避免多个来源同时影响任务生成。
- 后端仅允许执行单条 `SELECT` 查询，并限制最多返回 1000 行。
- SQL 返回字段会校验参数字段名，必须通过 `as` 别名与采集参数字段保持一致。
- SQL 导入模式会将 SQL 作为动态参数来源保存到 `wide_tables.scope_json.parameter_source`。
- 手动/文件导入模式会将固定参数行保存到 `wide_tables.scope_json.parameter_rows`。
- SQL 模式下，预览、保存定义、后端生成采集任务时都会重新执行已保存 SQL，生成当次参数行。

### 使用方式

1. 进入需求定义页面的“数据范围”区域。
2. 在“采集参数表”中切换到 `SQL 导入`。
3. 输入查询 SQL，字段别名需要和参数字段一致。
4. 点击 `执行导入`。
5. 导入成功后，下方参数表会显示 SQL 查询结果生成的参数行。

示例：

```sql
select distinct
  COMNAME as COMNAME,
  COMCODE as COMCODE
from IR_ADAS_COMPUTE_CONFIG
```

说明：

- 如果参数字段是 `COMNAME`、`COMCODE`，SQL 返回列也必须叫 `COMNAME`、`COMCODE`。
- 如果缺少字段，页面会提示缺少哪些参数列。
- SQL 来源是动态的；每次生成任务时会按已保存 SQL 重新查询。
- 手动/文件来源是静态的；保存后参数行固定不变。
