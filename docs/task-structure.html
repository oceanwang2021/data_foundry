任务结构与执行载体

当前数据模型同时覆盖按业务日期增量采集与按快照全量采集。二者共享同一套主模型，但在宽表行初始化、采集批次、任务组边界和补采能力上存在明确差异。

模式 A · 按业务日期增量采集
business_date / incremental
宽表示例
自动驾驶安全月度表
维度：company / city / biz_date
指标组
IG-MPI、IG-INCIDENT
两组指标分开执行
WideTableRow = 业务日期 × 其他维度组合
CollectionBatch
按业务日期组织执行波次，例如：
CB-WT-AD-SAFE-202501
→
TaskGroup
`partition_type = business_date`
例如：TG-WT-AD-SAFE-2025-01
→
FetchTask
按 `宽表行 × 指标组` 展开。
例如：2 行 × 2 组 = 4 个任务。
支持 BackfillRequest 历史补数 + 未来调度 任务组按业务日期聚合
模式 B · 按快照全量采集
none / full_snapshot
宽表示例
自动驾驶运营快照表
维度：company
指标组
IG-OPS-CORE
整组指标共享快照口径
WideTableRow = 非日期维度组合
CollectionBatch
按快照组织执行波次，例如：
CB-WT-AD-OPS-20260313
→
TaskGroup
`partition_type = full_table`
例如：TG-WT-AD-OPS-20260313
→
FetchTask
按 `宽表行 × 指标组` 展开。
例如：4 行 × 1 组 = 4 个任务。
不使用 business_date 按整表快照执行 不支持业务日期补采
共同执行层
FetchTask
任务最小业务单元
关键字段：`row_id`、`indicator_group_id`、`dimension_values`、`business_date / batch_id`
→
RetrievalTask
单指标检索任务
关键字段：`indicator_key`、`query`、`status`、`confidence`
→
NarrowIndicatorRow
单指标窄表载体
关键字段：来源链接、原文摘录、发布时间、指标逻辑说明、结果容器
→
WideTableRow 回填
按 `(row_id, indicator_key)` 回填
更新 `indicator_values`、`row_status`、`last_task_id`、`updated_at`
执行产物层： RetrievalTask、NarrowIndicatorRow、ExecutionRecord 与 ScheduleJob 共同承担查询明细、溯源信息、执行尝试与调度记录。它们服务于执行与治理，不改变主模型仍以 WideTableRow / CollectionBatch / TaskGroup / FetchTask 为主干的事实。
